Marketing Automation Platform

What's that now?

Marketing automation is a relatively new term, but we've been doing it for a long time.
  • 1997: WebProfiler
  • 2008: MyRegistrationForm
  • 2009: RMS
  • 2010: Email-o-Tron

WebProfiler: not so much automation, required a developer.

MyRegistrationForm: A never-released project that allowed users to create webforms and landing pages by picking a templated.

RMS: Response Management System. The MyRegForm front end layered on top of the WebProfiler framework. Originally built for BMC. It was easy to use for simple welcome » reg » thanks workflow. Harder for developers to make custom sites. Very strict templating.

Email-o-Tron: Original name for Magnolia

The original Emailotron header.

So what are they?

Magnolia

A tool for creating sites and emails

  • Fully customized PURLs
    • http://example.com/Alex.Luke
    • http://example.com/MrAwesome
  • No restrictions on HTML or design
    • Responsive design
    • Web fonts
    • <blink> and <marquee>
No technical restrictions on PURLs, but that doesn't mean that any purl should be used.

Thistle

The tracking database and lead distribution

  • Tracking at contact, touch and response level
  • Email, Direct Mail and Media campaigns
  • Lead distribution in any format and schedule
  • Can push leads into any system

Lots of data is collected that doesn't get reported on.

Previous integrations: Marketo, Eloqua, Salesforce, Siebel, Aprimo, WebEx, …

Recent features

  • Actual email sent numbers
  • Unsubscribes by touch
  • Direct Mail returns

Everlasting

The reporting dashboard

High level dashboard, not meant for deep introspection of data.

Reports are once an hour

We have tons mode data in Thistle if you want to see more.

Fully distributed architecture

A failure in any component doesn't affect any other parts of the system.

So why shouldn't we just use Marketo?

or Eloqua?

These tools don't need to be in competition. Our stack is generally easier to set up without any complicated service agreements. We're generally easier and more customizable for super complicated sites.
